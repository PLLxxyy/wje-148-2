# 拼车出行平台

一个全栈拼车出行平台，支持司机发布行程、乘客搜索和加入拼车、申请审核、双向评价等功能。

## 技术栈

- **前端：** Vite + React 18 + TypeScript（端口 5173）
- **后端：** Express + TypeScript + better-sqlite3（端口 3000）
- **认证：** JWT + bcryptjs
- **启动方式：** concurrently 同时启动前后端

## 启动方式

```bash
# 安装所有依赖（根目录 + client）
npm run install:all

# 启动前后端（自动并发启动）
npm run dev
```

启动后访问 http://localhost:5173

## 功能说明

### 用户系统
- 注册（选择司机/乘客身份）、登录、个人资料编辑
- JWT 令牌认证，7天有效期

### 行程管理（司机端）
- 发布新行程：填写出发地、目的地、出发时间、车型、可载人数、每人费用
- 行程管理：查看所有已发布行程，审核乘客申请（通过/拒绝）
- 完成行程、取消行程（取消后自动通知已加入的乘客）
- 满员自动关闭，拒绝剩余待审核申请

### 行程搜索与加入（乘客端）
- 首页搜索：按出发地、目的地搜索，按日期筛选
- 行程卡片：显示司机头像、路线、时间、剩余座位、费用
- 加入拼车：填写上车点和乘车人数
- 行程详情：查看司机信息、已加入乘客、评价记录

### 我的行程
- 即将出发 / 历史记录 双Tab切换
- 司机查看自己发布的行程，乘客查看参与的行程

### 评价系统
- 行程完成后，乘客可评价司机，司机可评价乘客
- 1-5星评分 + 文字评价
- 用户评分自动计算更新

### 通知系统
- 实时通知：申请被通过/拒绝、行程被取消、行程完成提醒
- 未读消息提醒，支持全部标记已读

## 注意事项

1. 数据库文件自动创建在 `server/data/carpooling.db`，首次启动自动建表并插入测试数据
2. 已预置5条示例行程数据供测试
3. 前端通过 Vite 代理 `/api` 请求到后端 3000 端口
4. 样式全部写在 `index.html` 的 `<style>` 标签中，无外部 CSS 文件
5. 取消行程会通知所有已加入的乘客
6. 行程满员后自动关闭新申请
